Proposal for a cognitively ergonomic human-oriented IDE.

This currently exists as a theorycrafting, hypothetical IDE of the best future. Coding could look and feel very different from how we traditionally code today, this tool is capturing it extensive research & cognitive tools at [meta-lang]() into an integrated development environment, designed for human cognition.

## "Cogware" 

This program might be the first of its kind; [Cogware]() a fundamental approach to the programmability of computers, not as abstract bendable software, but as cognitive wear, between the hardware and the human interaction. but also, emphasis on cog, many working parts. what if programming came more in micropackages that we could assemble based on our needs? we already see loosely developed models of this in our real world, the question is how do we turn it into a fundamental part of the design so it drives development and human usage of computation programmability?
