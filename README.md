# ^m
Project meta-ide (^m).

<sup>Copyright (C) 2026 Metasystems </sup>

Proposal for a cognitively ergonomic human-oriented IDE.

This currently exists as a theorycrafting, hypothetical IDE of the best future. Coding could look and feel very different from how we traditionally code today, this tool is capturing it extensive research & cognitive tools at [meta-lang]() into an integrated development environment, designed for human cognition.

## Product vision

One of the most problematic parts of coding, isn't code itself; it is the process and tools we use to author code. A perfect tool will most likely never solve all of our issues, but good tools make problem-solving, less painful.

- __An instrument which allows the engineer to Experiment, Theorycraft, or Build real-world systems, in the same enviornment.__ Providing enviornments where you can both: experiment and learn new ways of building systems, aswell as building real-world systems that you can understand, track, visualize and share with others.
- __Sticking to plain-text format, visualized with dynamic graphics.__ Everything this tool fundamentally integrates, is a _file/plain-text, non-proprietary formats, open-source._ Sticking to the pre-established idea of **"Source code should be plain-text, ambiguous, and non-biased. Interaction should be biased, specialized and graphical.".**
- __Fundamental redesign of code authoring.__ The tool's vision is _"if you can replace any feature of this tool, with another IDE, or a plain-text editor, it hasn't solved a real issue."_

## "Cogware" 

This program might be the first of its kind; [Cogware]() a fundamental approach to the programmability of computers, not as abstract bendable software, but as cognitive wear, between the hardware and the human interaction. but also, emphasis on cog, many working parts. what if programming came more in micropackages that we could assemble based on our needs? we already see loosely developed models of this in our real world, the question is how do we turn it into a fundamental part of the design so it drives development and human usage of computation programmability?
