The structure of this meta ide, is not focused on exciting new technologies, rather, the purpose of any cogware is to zero in on highly effective isolated components of technology, that can act as input/output transformation functions to be used with other systems. in this sense, technology's become compostable and modular. this effectively reduces static skeletal systems, for the human operator, but it increases an enormous amount of system design for the human engineer. 

This is what leads us into the design philosophy of the meta IDE.

this is where we have ideas for, common templates for directories and file structures. interaction prompts for maintaining consistency across a code base, and several ideas, at the interaction level. which is often completely ignored in modern tooling, possibly explaining why the systems we end up creating our fundamentally unmanageable and unreadable, the tools we use, feel like static industrial products, when they should feel like instruments where we can compose parts to create a new things. 

> "Add new main module at 'api/core'"

(Internally simply creates a new main.js file, with module capabilities. but for the human engineer, this fundamentally reduces cognitive weight, and increases reusability of interaction, ensuring a stable code base.)
