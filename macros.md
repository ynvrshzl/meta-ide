# Macros...
These terms are interchangeable, macros, commands, scripts, utils, patterns, templates, reusables, composables, etc. 

They all describe the same concept, the ability to reuse structure, across systems. 

although these words are used to describe the same concept, we don't imagine them as being the same shape or function. for example a command is not associated with a repeatable action, although that's apparently what a script is. and a template is associated more with static text, but I can apply to entire structures, which includes macro automation. in this sense, we tap into a powerful concept of the meta ide, names are shapes. it's important to choose the proper name for the shape we are describing.

In this sense, we are able to reuse workflows, patterns, build steps, scripts and structures and templates, across our systems. this is not only incredibly time-saving, but it reduces energy wasted having to retype the same patterns, build the same workflows, never generating new insight or understanding, this is fundamentally an automation problem and that is fundamentally what computers are for!
