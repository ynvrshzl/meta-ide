# What is the first thing the programmer should see, when their instrument begins?

Sample buttons like "Create new project" and "Open recent projects" are great for utility, but this is the first screen the programmer will likely see hundreds of times; so it needs to provide dynamic, context-sensitive information about the instrument, and the programmer's code.

Some candidates include...
- News feed of community driven projects using this instrument
- Programmer's recent activity (not merely static files/folders, but headmaps like github.)
- Syncrhonized projects/tasks
- "Jump back in" to recent projects or workflows.

These can be modularly assembled/disabled, as with all [cogware](#) ideas.
